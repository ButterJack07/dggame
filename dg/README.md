# 🏰 dg — 帝国时代风格资源管理游戏

基于 Canvas 等距视角的 50×50 地图资源管理与建造游戏，单 HTML 文件。

## 快速开始

直接打开 `dg/game.html` 即可游玩，无需构建或安装。

## 操作

| 操作 | 方式 |
|------|------|
| 平移地图 | 鼠标拖拽 / 触摸拖动 / 方向键 |
| 缩放 | 滚轮 / 双指捏合 |
| 选中地块 | 点击地块 |
| 建造建筑 | 点击左下方面板中的建筑，再点击地图上有效位置 |
| 取消建造 | 右键 / `Esc` |
| 刷新地图 | 按 `R` |
| 100× 采集 | 点击右上角 🐇 按钮 |

## 建筑

9 种建筑，其中 6 种为装饰性建筑，3 种为功能性资源建筑：

| 建筑 | 类型 | 功能 |
|------|------|------|
| 城镇中心 🏛 | 装饰 | — |
| 兵营 ⚔ | 装饰 | — |
| 马厩 🐴 | 装饰 | — |
| 教堂 ⛪ | 装饰 | — |
| 市场 🏪 | 装饰 | — |
| 房屋 🏠 | 装饰 | — |
| 伐木场 🪵 | 资源 | 采集 3 格范围内木头 |
| 采矿场 ⛏️ | 资源 | 采集 3 格范围内黄金 |
| 磨坊 🌾 | 资源 | 采集 3 格范围内食物 |

限制：不能建在水面或已有建筑的地块上。

## 资源系统

三种资源，各自绑定特定地形，且**可耗尽**：

| 资源 | 地形 | 图标 | 初始储量 |
|------|------|------|----------|
| 🪵 木头 | 森林 | 5 档松树图标 | 60–150 |
| ⛏️ 黄金 | 金矿 | 5 档晶体图标 | 30–80 |
| 🌾 食物 | 浆果丛 | 5 档浆果图标 | 30–80 |

- 每个资源地块初始显示 **5 个图标**，消耗 20% 消失一个
- 图标按固定版式散落在地块菱形顶面上
- 选中资源地块时显示小型进度条
- 木材耗尽可能导致森林退化 → 金矿耗尽 → 晶体和对象消失 → 变为草地

## 架构

项目为单一 `game.html` 文件，按功能分区：

### 1. 常量与配置 (行 ~157–217)

```
TILE_W=96, TILE_H=48   — 菱形瓦片尺寸
TILE_H_MAX=60           — 地形最大隆起
MAP_N=50                — 地图边长
cam = { x, y, zoom }    — 相机状态
terrainTypes[]          — 5 种地形定义（颜色、明暗面）
buildingTypes[]         — 9 种建筑列表
buildingData{}          — 3 种资源建筑的采集参数（range, rate）
```

### 2. 地图生成 `generateMap()` (行 ~275–443)

流程：
1. **平滑高度图** — 随机山丘 + 平滑插值 + 归一化
2. **地形分配** — 低洼→水，中高→草地
3. **森林簇** — 8-15 簇 BFS 扩散，每簇 20-50 片
4. **沿海沙滩** — 临水地块替换
5. **金矿簇** — 3-8 簇*5 尝试，每簇 1-5 连片
6. **浆果丛** — 5-10 簇，每簇 2-5 片

每个地块结构：`{ terrain, height, object, resource }`

### 3. 坐标系统

- `mapToScreen(r, c, z)` — 行列 → 屏幕像素（等距投影）
- `screenToMap(px, py)` — 屏幕像素 → 浮点行列
- `getTileFromScreen(mx, my)` — 点击检测，在 ±2 邻域内找最近菱形命中

### 4. 渲染管线 `render()` (行 ~1620)

每帧执行顺序：

```
clearRect → 天空渐变背景
gatherTick()              ← 每 10 帧执行资源采集
buildingLabels = []       ← 重置标签列表
ctx 缩放 + 平移
绘制地下基座（防悬空）
视锥裁剪 → 构建可见瓦片列表
瓦片按 (r + c + height*0.01) 深度排序
  ↓
for each 瓦片:
  drawTile()              ← 地形、墙壁、选中高亮、资源图标、建筑物体
  if 选中且为资源建筑:
    绘制采集范围高亮      ← 融入同一个循环，保证深度正确

drawGhost()               ← 建造预览虚影
draw buildingLabels[]     ← 所有建筑名字标签
ctx.restore()
绘制 UI:
  选中信息面板            ← 坐标/地形/资源/采集进度条
  资源面板（右下角）      ← 木头/黄金/食物总量
```

### 5. `drawTile(r, c, cell)` (行 ~498)

单个瓦片绘制流程：

```
1. 计算菱形顶点 (top/bot/left/right)
2. 绘制顶面（地形颜色）
3. 水面 → 仅邻边网格线
4. 非水面 → 四边描边
5. 高度 > 0 → 右侧壁（暗面）+ 左侧壁（亮面）
6. 选中 → 金色发光描边
7. 有资源 → 5 档图标散落 + 进度条（仅选中）
8. 有物体 → drawObject()
```

### 6. `drawObject(x, y, obj)` (行 ~648)

每种建筑有独立的 Canvas 绘制代码：

| 建筑 | 绘制方式 |
|------|----------|
| 城镇中心 | 三层墙体 + 窗口 + 尖顶 + 城垛 + 旗帜 + 阳台 |
| 兵营 | 双斜面建筑 + 门 + 屋顶 + 旗杆 |
| 马厩 | 长条形 + 屋顶 + 马棚门 |
| 教堂 | 双塔 + 玫瑰窗 + 尖顶 |
| 市场 | 开放摊位 + 顶棚 + 商品 |
| 房屋 | 小型双斜面 + 烟囱 |
| 金矿 | 地面阴影 + 多面晶体簇 |
| 伐木场/采矿场/磨坊 | 小型建筑 + 各自特征 |

### 7. 采集逻辑 `gatherTick()` (行 ~1542)

```
每 10 帧执行:
  遍历所有地块:
    如果地块有资源建筑（gatherRange 属性）:
      在 Manhattan 距离 ≤ range 范围内查找匹配资源:
        进度 += gatherRate (×100 如果 speed100x)
        进度 ≥ 1 → 收获 1 单位 + 减少源资源量
        资源耗尽 → 清除 resource / 退化地形
        break scanLoop (每 tick 只采一个来源)
```

关键变量：
- `gatherRate`: 基础采集速度（0.03~0.04/10帧）
- `gatherProgress`: 当前采集进度（累计至 ≥1 收获）
- `resourcesCollected`: 该建筑总收获计数
- `resources{}`: 全局资源总量

### 8. 资源图标系统 (行 ~552)

- `totalIcons = 5` — 每格最多 5 个图标
- `count = min(5, floor(pct × 5) + 1)` — 图标数随储量线性减少
- 固定版式 `layouts[]` — 5 个位置统一分布在菱形顶面
- 三种图标绘制：
  - **木材**: 三层叠加松树（深绿→亮绿）+ 树干
  - **黄金**: 多面切割晶体（底影 + 高光 + 阴影面，与金矿风格一致）
  - **食物**: 三颗浆果簇 + 高光 + 茎 + 绿叶

### 9. 交互与事件 (行 ~247 起)

| 事件 | 处理 |
|------|------|
| 建筑面板点击 | 切换 placementMode |
| mousedown | 开始拖拽 / 点击放置 |
| mousemove | 更新拖拽 / ghost 预览 |
| mouseup | 结束拖拽 / 点击选中 |
| contextmenu | 取消建造 |
| wheel | 缩放 |
| touch | 拖拽 + 双指缩放 |
| keydown | 方向键平移 / R 刷新 / Esc 取消 |

## Debug 提示

- **100× 速度开关**: 右上角 🐇 按钮，调试采集系统时使用
- **选中信息**: 点击地块查看坐标、地形、高度、建筑、资源储量、采集进度
- **采集范围**: 点击资源建筑显示 3×3 曼哈顿范围高亮（绿色=有资源，灰色=空）
- **日志**: 所有 UI 状态通过 `buildingLabels[]` 传递，非阻塞
- **地图刷新**: 按 `R` 重新生成地形和资源，重置所有状态
- **深度排序问题**: 资源图标和范围高亮必须在 `drawTile` 内部绘制，不能后置，否则与 3D 墙壁产生 alpha 混合黑块
- **资源耗尽**: 森林→草地（仅地形变化），金矿→草地（地形 + 清除 object）
- **采集节奏**: 每 10 帧只采 1 个资源点（`break scanLoop`），避免一帧内清空整片资源
